# README: Анализ логов безопасности и выявление аномалий

## Описание проекта

Проведен анализ 30-дневного среза обезличенных логов корпоративной инфраструктуры (события аутентификации, активность на рабочих станциях, сетевые соединения). Цель - найти следы реальных атак и предложить систему детектирования угроз.

## Задачи

1. Исследовательский анализ данных (EDA)
2. Поиск подозрительной активности 
3. Разработка правил детектирования
4. Построение механизма выявления аномалий (DBSCAN)

## Структура

- `solution.ipynb` - Jupyter Notebook с полным анализом и кодом
- `README.md` - текущий файл
- `requirements.txt` - требования для запуска 

## Методология

### 1. Исследовательский анализ данных

Проведен первичный анализ структуры данных: распределение по источникам логов (network, endpoint, auth), по пользователям, отделам и времени. Выявлены основные закономерности: пик активности приходится на 6:00-18:00 UTC, доминируют network-события. Обнаружены проблемы качества: пропуски в полях geo_country, file_size_mb, дубликаты event_id.

### 2. Поиск подозрительной активности

Найдено 5 аномальных инцидентов:

| № | Пользователь | Инцидент | Риск |
|---|-------------|----------|------|
| 1 | U0042 | Брутфорс: 34 неудачные попытки из 3 стран → успешный вход | Критический |
| 2 | U0056 | Инсайдер: USB-копирование в выходной день (воскресенье) | Высокиий |
| 3 | U0017 | Всплеск активности: 5.3 GB внешнего трафика за 4 дня | Критический |
| 4 | U0056 | Утечка 6.13 GB на внешний IP 104.18.32.155 | Критический |
| 5 | WS-ENG-017 | Аномальный всплеск внешнего трафика (5.3 GB в 22:00) | Высокий |

Для каждого инцидента указаны: причины подозрительности, подтверждающие данные, уровень риска и дополнительные данные для SOC.

### 3. Система детектирования

Разработано 5 правил обнаружения угроз с описанием логики, примером реализации на Python и разбором ложноположительных срабатываний:

1. Брутфорс по неудачным входам из разных стран
2. Всплески внешнего трафика (статистический порог)
3. Инсайдерская активность (USB + нерабочее время)
4. Цепочка "архив -> внешний трафик"
5. Всплеск активности пользователя (аномальный рост)

### 4. Выявление аномалий (DBSCAN)

Для выявления аномалий выбран метод DBSCAN (кластеризация на основе плотности). Выбор обоснован отсутствием предположений о распределении данных, устойчивостью к выбросам и способностью находить кластеры произвольной формы.

**Использованные признаки (9):**

- `total_events` - общее количество событий за день (выявляет внезапные всплески активности)
- `external_traffic_mb` - объем внешнего трафика (прямой индикатор утечки данных)
- `external_connections` - количество внешних соединений (признак массовой отправки)
- `archive_count` - количество созданных архивов (подготовка к утечке)
- `usb_events` - количество USB-подключений (инсайдерская активность)
- `powershell_count` - количество запусков PowerShell (инструмент для атак)
- `file_downloads` - количество скачанных файлов (систематический сбор данных)
- `failed_logins` - количество неудачных входов (брутфорс)
- `unique_countries` - количество разных стран входа (аномальная география)

Все признаки выбраны на основе реальных аномалий, обнаруженных в Задании 2.

**Результаты DBSCAN:**

- Всего записей (пользователь-день): ~2800
- Найдено аномалий: ~30 (около 1%)
- Подтверждены все известные инциденты: U0056, U0017, U0042
- Обнаружены новые подозрительные пользователи, требующие дополнительной проверки

## Выводы

В ходе работы подтверждено, что в логах присутствуют следы реальных атак: брутфорс, утечка данных, инсайдерская активность. Разработанные правила детектирования и модель DBSCAN позволяют автоматически выявлять как известные, так и новые аномалии в потоке событий. Результаты могут быть использованы для настройки системы мониторинга безопасности (SOC).
